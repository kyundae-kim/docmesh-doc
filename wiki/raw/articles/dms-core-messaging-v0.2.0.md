# 메시지 정의서

## 1. 문서 목적

본 문서는 현재 DMS SDK 범위에서 메시지 기반 외부 계약이 존재하는지 여부와 그 경계를 정의합니다.

## 2. 현재 제품 경계

현재 코드 기준 DMS SDK는 다음 기능만 제공합니다.
- 문서 업로드
- 문서 메타데이터 조회
- 문서 본문 전체 조회
- 문서 본문 스트리밍 조회
- 문서 삭제
- 상태 점검
- 환경 기반 조립 및 종료 정리

현재 공개 API에는 다음이 포함되지 않습니다.
- 메시지 브로커 발행
- 메시지 브로커 구독
- 이벤트 스키마 직렬화/역직렬화
- 비동기 작업 큐 연계
- 웹훅 전송

## 3. 메시지 계약 판단

현재 DMS SDK는 동기식 Python SDK이며, 코드베이스 안에 별도의 메시지 모델이나 메시지 전송 인터페이스가 존재하지 않습니다.
따라서 현재 버전에서 독립된 메시지 계약 문서는 "적용 대상 없음" 상태로 유지합니다.

## 4. 관련 운영 사실

환경 기반 조립 과정에서 `docmesh-py-core` 설정 검증을 통과하기 위해 런타임 환경에 `NATS_SERVERS` 같은 추가 서비스 설정이 필요할 수 있습니다.
그러나 이는 공통 설정 로더의 검증 요구사항일 뿐이며, 현재 DMS SDK 자체가 NATS 메시지를 발행하거나 구독한다는 의미는 아닙니다.

## 5. 향후 문서 확장 조건

다음 중 하나가 코드에 추가되면 본 문서를 실제 메시지 계약 문서로 확장해야 합니다.
- 이벤트 발행 API
- 소비자/구독자 API
- 메시지 payload 모델
- 재시도/중복처리/순서보장 규칙
- 외부 브로커 주제명 또는 큐 이름 계약
